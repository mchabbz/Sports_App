Sports Data Management System

Explore Top Players Across MLB, NBA, and NFL

This web application organizes player performance data from three major sports leagues: Major League Baseball, the National Basketball Association, and the National Football League. The data was loaded from CSV files into a SQLite database and is displayed through dynamic Flask pages using SQLAlchemy ORM queries.

Users can browse players by league, search for records, filter by team, sort by performance metrics, and view detailed information for each player. Registered users can also create, update, and delete records based on authorization rules.

What This Application Demonstrates

Database Design

The app uses Flask-SQLAlchemy models for sports, players, and users. Player records are connected to sports through relationships.

Dynamic Data Display

Player data is not hardcoded. It is queried from the database and rendered dynamically using Jinja templates.

Search, Filter, and Sort

Each sport has its own search page, while the home page provides a global search across MLB, NBA, and NFL.

Create, Update, Delete

Logged-in users can add player records. Users can manage records they created, while admin users can manage all records.

User Accounts

Registration and login allow the application to connect created records to user accounts and protect sensitive routes.

Light and Dark Mode

The interface includes a smooth theme switcher so users can choose between a light theme and a dark theme.

About the Metrics

Each player record includes four numeric performance metrics from the CSV files. These generic metric columns make the application flexible enough to display different sports data while keeping a consistent table structure across MLB, NBA, and NFL.

The ID column shown in tables is a database-generated primary key. It is used by the application to view, edit, and delete individual records. It does not represent a player ranking.
